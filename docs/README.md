# Docs Layout

Use `docs/` as the home for repository documentation that is not part of the Android app source tree.

## Folders

- `docs/project/`
  - Canonical planning, architecture, decisions, open questions, and workflow docs for the project.
- `docs/development/`
  - Local setup and day-to-day development environment guides.

## Root-level docs that stay at the repo root

- `README.md`
  - Project overview and quick-start summary.
- `AGENTS.md`
  - Repo guardrails, product scope, and agent-specific instructions.
